# Topics

Index notes per theme. Each Topic aggregates related notes scattered across the vault (People, Library, Projects, Explorations) under a single entry point.

**File naming**: `<topic>.md` in kebab-case (e.g. `ai.md`, `networking.md`, `startup-growth.md`).

**Create a Topic when**: you have 5+ notes on the same theme spread across different folders and you want a single entry point for that theme.

A good Topic has:
- Short intro: what this theme means to you
- Sections grouping linked notes by sub-theme
- Open questions / current state of thinking
- Links to related Topics

**Topic ≠ Exploration**: an Exploration has a guiding question and an end. A Topic is evergreen — it grows over time without expiring.
